# quiz-stats

034 퀴즈 게임의 문제 데이터 통계를 분석하고 보고한다.

## 입력
- `$ARGUMENTS` : 분석 모드(선택)
  - 비어 있거나 `all` : 전체 통계
  - 카테고리명(`여행` / `음식` / `일상` / `미술/예술`) : 해당 카테고리만

## 대상 파일
- `034/src/data/questionBank.js`
- `034/src/core/constants.js` (카테고리 목록 확인용)

## 실행 규칙

### 1단계 — 문항 로드
`QUESTIONS` 배열을 파싱해 전체 문항을 로드한다.

### 2단계 — 기본 통계 계산

**전체 요약**
- 총 문항 수
- 카테고리별 문항 수 및 목표(10문항) 대비 달성률
- 난이도별 문항 수 (`easy / medium / hard`)

**정답 인덱스 분포**
- `correctAnswer` 0~3 각각의 빈도 및 비율
- 이상적 균형(각 25%)과의 편차

**문항 품질 지표**
- 평균 `question` 길이 (글자 수)
- 평균 `explanation` 길이
- `options` 평균 글자 수

### 3단계 — 카테고리 상세 통계
각 카테고리에 대해:
- 문항 수 / 목표 달성률
- 난이도 분포 (easy/medium/hard 개수)
- 정답 인덱스 분포
- 가장 짧은 해설 / 가장 긴 해설 ID

### 4단계 — 이상 탐지
- 목표(10문항)보다 부족한 카테고리 → 추가 필요 수 표시
- 난이도 편중: 한 난이도가 카테고리 내 70% 초과이면 경고
- 정답 인덱스 편중: 특정 인덱스 40% 초과이면 경고

## 출력 템플릿

```
=== 034 퀴즈 데이터 통계 ===
분석 범위  : [전체 / 카테고리명]
총 문항 수 : N개

[카테고리별 현황]
카테고리   | 문항 수 | 목표 | 달성률 | 부족
-----------|---------|------|--------|-----
여행       |       N |   10 |   N%   |  N
음식       |       N |   10 |   N%   |  N
일상       |       N |   10 |   N%   |  N
미술/예술  |       N |   10 |   N%   |  N

[난이도 분포]
easy: N개 (X%)  medium: N개 (X%)  hard: N개 (X%)

[정답 인덱스 분포]
0번: N개 (X%)  1번: N개 (X%)  2번: N개 (X%)  3번: N개 (X%)

[경고]
- 없음 / 상세 내용

[권고 사항]
- 부족한 카테고리·난이도 조합 목록
```

## 규칙
- 경고가 없으면 `✓ 데이터 균형 양호` 메시지를 출력한다.
- 숫자는 소수점 1자리까지만 표시한다.
