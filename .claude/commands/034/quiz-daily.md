# quiz-daily

퀴즈 데이터를 점검하고 새 문항을 추가하는 7단계 일일 루틴을 순서대로 실행한다.
각 단계 실패 시 즉시 중단하고 오류를 보고한다.

## 입력
- `$ARGUMENTS` : 추가할 카테고리와 난이도 (`<카테고리> <난이도>`, 선택)
  - 예: `여행 medium`
  - 비어 있으면 부족한 카테고리·난이도를 자동 선택

## 대상 파일
- `034/src/data/questionBank.js`
- `034/src/core/constants.js`

---

## 단계별 실행 규칙

### 1단계 — 파일 구조 읽기
`034/src/data/questionBank.js`와 `034/src/core/constants.js`를 읽는다.
- `QUESTIONS` 배열의 전체 구조, 필드명, 값 형식을 확인한다.
- 카테고리 목록(`CATEGORIES`)과 키(`RANKING_STORAGE_KEY`)를 확인한다.
- **실패 조건**: 파일이 없거나 `QUESTIONS`가 배열이 아닌 경우 → 즉시 중단.

### 2단계 — 현재 문제 개수와 분포 확인
- 총 문항 수를 집계한다.
- 카테고리별 문항 수를 집계한다.
- 난이도별(easy/medium/hard) 문항 수를 집계한다.
- **실패 조건**: 총 문항 수가 0이거나 카테고리 필드가 없는 문항 존재 시 → 즉시 중단.

### 3단계 — 카테고리별 부족 분석
카테고리 목표: 각 카테고리 10문항.
- 각 카테고리별 부족 수(`목표 - 현재`)를 계산한다.
- 난이도 균형 기준: easy 30%, medium 50%, hard 20%를 이상으로 한다.
- 부족한 카테고리와 필요한 난이도를 우선순위 순으로 나열한다.
- `$ARGUMENTS`로 카테고리와 난이도가 지정되었으면 해당 값을 사용한다.
- **실패 조건**: 모든 카테고리가 목표를 달성했고 `$ARGUMENTS`가 비어 있으면 → "추가 불필요"를 보고하고 종료.

### 4단계 — 중복 체크
3단계에서 선택한 카테고리·난이도에 이미 존재하는 문항의 `question`을 수집한다.
새로 추가할 문항이 기존 문항과 80% 이상 유사하지 않아야 한다.
- **실패 조건**: 추가하려는 문항이 기존 문항과 중복(동일 또는 80% 이상 유사) → 즉시 중단 후 중복 문항 ID 보고.

### 5단계 — 문제 추가 및 형식 검증
다음 형식으로 문항 1개를 생성하고 `questionBank.js`에 추가한다:

```javascript
{
  id: <기존 최대 ID + 1>,
  category: "<카테고리>",
  difficulty: "<난이도>",
  question: "문항 본문",
  options: ["선택지1", "선택지2", "선택지3", "선택지4"],
  correctAnswer: <0|1|2|3>,
  explanation: "해설"
}
```

추가 후 다음을 검증한다:
- `id` 유일성
- `options` 4개, 중복 없음
- `correctAnswer`가 0~3 범위
- `question`이 물음표로 끝남
- `explanation`이 비어 있지 않음
- 모호 표현(`가장`, `최초`, `최대`) 사용 시 기준 명시 여부

**실패 조건**: 위 검증 항목 중 하나라도 미통과 → 추가한 문항을 롤백하고 즉시 중단.

### 6단계 — 백업
추가 완료 후 `questionBank.js`의 현재 내용을 다음 경로에 복사한다:
```
034/src/data/backup/questionBank_<YYYY-MM-DD>.js
```
- 백업 디렉터리(`034/src/data/backup/`)가 없으면 생성한다.
- **실패 조건**: 백업 파일 쓰기 실패 → 경고를 출력하고 계속 진행(중단하지 않음).

### 7단계 — 실행 결과 보고
아래 템플릿으로 최종 보고서를 출력한다:

```
=== 034 퀴즈 일일 루틴 실행 보고 ===
실행 일시     : YYYY-MM-DD HH:MM
완료 단계     : N / 7

[단계별 결과]
1단계 파일 구조 읽기         : ✓ PASS / ✗ FAIL — <사유>
2단계 문제 개수·분포 확인    : ✓ PASS / ✗ FAIL — <사유>
3단계 부족 분석              : ✓ PASS / ✗ FAIL — <사유>
4단계 중복 체크              : ✓ PASS / ✗ FAIL — <사유>
5단계 문제 추가·형식 검증    : ✓ PASS / ✗ FAIL — <사유>
6단계 백업                   : ✓ PASS / ⚠ WARN — <사유>
7단계 보고 생성              : ✓ PASS

[추가된 문항]
ID: N | 카테고리: X | 난이도: X
문항: "..."
정답: options[N] = "..."

[현재 데이터 현황]
총 문항 수: N개
카테고리   | 현재 | 목표 | 달성률
-----------|------|------|-------
...

최종 결과: 성공 / 실패 (N단계에서 중단)
```

## 오류 보고 형식
단계 실패 시 다음 형식으로 즉시 출력하고 중단한다:
```
[중단] N단계 실패: <오류 내용>
조치 필요: <권고 사항>
```
