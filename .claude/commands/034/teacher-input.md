# teacher-input

학생들의 퀴즈 성적을 입력하고 `teacher_scores.json`에 저장한다.

## 입력
- `$ARGUMENTS` : `<이름> <점수> [카테고리] [모드]`
  - `이름` : 학생 이름 (필수)
  - `점수` : 숫자 (필수, 0 이상)
  - `카테고리` : `여행 | 음식 | 일상 | 미술/예술 | 전체` (선택, 기본값: `전체`)
  - `모드` : `full | category | speed` (선택, 기본값: `full`)
  - 예: `홍길동 85 여행 category`

## 데이터 파일
- `034/teacher_scores.json` — 성적 데이터 저장소

## 저장 구조
```json
{
  "students": [
    {
      "name": "홍길동",
      "records": [
        {
          "score": 85,
          "category": "여행",
          "mode": "category",
          "date": "2026-05-06T10:00:00.000Z"
        }
      ]
    }
  ]
}
```

## 실행 규칙

### 1단계 — 입력 파싱
`$ARGUMENTS`를 파싱해 이름, 점수, 카테고리, 모드를 추출한다.
- 이름 또는 점수가 없으면 → 즉시 중단, 사용법 안내.
- 점수가 숫자가 아니거나 음수이면 → 즉시 중단.

### 2단계 — 데이터 파일 로드
`teacher_scores.json`이 있으면 읽고, 없으면 빈 구조로 초기화한다.

### 3단계 — 기존 학생 확인
같은 이름의 학생이 있으면 해당 학생의 `records`에 추가한다.
없으면 새 학생 항목을 생성한다.

### 4단계 — 저장 및 보고
```
✓ 성적 입력 완료
학생: 홍길동 | 점수: 85점 | 카테고리: 여행 | 모드: category | 날짜: 2026-05-06
해당 학생 총 기록 수: N개
```

## 규칙
- 같은 학생이 같은 카테고리에 여러 기록을 가질 수 있다(중복 허용).
- 날짜는 입력 시점의 ISO 8601 형식으로 자동 기록한다.
