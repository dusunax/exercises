# quiz-check

`src/data/questionBank.js`의 모든 문제에 대해 정답 정확성을 검증한다.

## 입력
- `$ARGUMENTS` : 검증할 카테고리명(선택). 비어 있으면 전체 검증.

## 대상 파일
- `034/src/data/questionBank.js`

## 실행 규칙

### 1단계 — 파일 로드 및 파싱
`QUESTIONS` 배열을 읽어 전체 문항을 메모리에 로드한다.
`$ARGUMENTS`가 있으면 해당 카테고리만, 없으면 전체를 대상으로 한다.

### 2단계 — 구조 검증 (각 문항)
다음 필드가 모두 존재하고 올바른 타입인지 확인한다:

| 필드 | 타입 | 조건 |
|---|---|---|
| `id` | Number | 양의 정수, 중복 없음 |
| `category` | String | `'여행' \| '음식' \| '일상' \| '미술/예술'` 중 하나 |
| `difficulty` | String | `'easy' \| 'medium' \| 'hard'` 중 하나 |
| `question` | String | 비어 있지 않음, `?` 또는 `는?`으로 끝남 |
| `options` | Array | 길이 정확히 4, 빈 항목 없음, 4개 중 중복 없음 |
| `correctAnswer` | Number | `0 \| 1 \| 2 \| 3` 중 하나 |
| `explanation` | String | 비어 있지 않음 |

### 3단계 — 정답 정확성 검증
각 문항의 `correctAnswer` 인덱스가 가리키는 `options[correctAnswer]` 값이 `explanation` 내용과 논리적으로 일치하는지 확인한다.

- 일치하면 `PASS`
- 불일치 또는 불확실하면 `NEEDS_REVIEW` + 사유 제시

### 4단계 — 중복 문항 탐지
`question` 문자열이 동일하거나 80% 이상 유사한 쌍을 탐지해 보고한다.

### 5단계 — 정답 인덱스 편향 검사
전체 `correctAnswer` 분포에서 특정 인덱스(0~3)가 40% 초과이면 편향 경고.

## 출력 템플릿

```
=== 034 퀴즈 정답 정확성 검증 보고서 ===
검증 범위  : [전체 / 카테고리명]
검증 일시  : [실행 시각]
총 문항 수 : N개

[구조 오류]
- 없음 / 문항 ID 목록 + 오류 내용

[정답 검증]
ID  | 카테고리 | 난이도 | 상태          | 사유
----|----------|--------|---------------|------
...

[중복 의심 쌍]
- 없음 / (ID X, ID Y) 유사도 XX%

[정답 인덱스 분포]
0번: N개 (X%)  1번: N개 (X%)  2번: N개 (X%)  3번: N개 (X%)
편향 경고: 없음 / [인덱스 N이 X%로 초과]

최종 판정: PASS / FAIL (N개 항목 수정 필요)
```

## 규칙
- 구조 오류가 1개라도 있으면 즉시 `FAIL` 판정 후 상세 목록 출력.
- `NEEDS_REVIEW` 항목은 수정 권고안을 함께 제시한다.
- 모든 항목이 통과하면 `✓ 전체 PASS` 메시지를 출력한다.
