# quiz-validate

사용자는 퀴즈 데이터의 품질을 점검하기 위해 아래 지침을 따른다.

## 입력
- `$ARGUMENTS` : 검증할 카테고리명(선택). 값이 있으면 해당 카테고리만 검증.
- `$ARGUMENTS`가 비어 있으면 전체 문제(`전체`)를 검증.

## 대상 파일
- `src/data/questionBank.js`

## 실행 규칙
1. `const QUESTIONS` 배열을 로드한다.
2. 카테고리 필터를 적용한다.
   - `$ARGUMENTS`가 비어 있지 않으면 `category === $ARGUMENTS` 만 대상.
   - 비어 있으면 모든 문제를 대상.
3. 대상 문제 각각의 `question`, `explanation`, `category`, `difficulty`, `correctAnswer`를 확인한다.
4. 아래 모호 표현이 포함된 문항을 탐지해 보고한다.
   - `가장`, `최초`, `최대`
5. 단순 탐지만 하지 말고, **어떤 기준을 명시해야 통과되는지**를 같이 제시한다.

## 모호 표현의 판정 기준(반드시 출력)
- `가장`
  - 무엇과 비교해 가장인지(예: 세계/아시아/국내, 1990년~현재 등)
  - 수치인지, 순위인지, 빈도인지 기준 정의
  - 최신 근거 시점(연도) 또는 출처
- `최초`
  - 최초의 대상이 되는 사건/개념 정의(최초로 기록된 것인지, 최초의 공식 인정인지)
  - 출처(기관/문헌)와 시점(연도/세기)
- `최대`
  - 측정 단위와 범위(면적, 길이, 개수, 시간 등)
  - 대상 범위(국가 전체, 공식 통계 대상, 기간)
  - 집계 기준(동일 단위 환산 방식 포함)

## 출력 템플릿
1. 대상 범위
   - 전체 / 특정 카테고리명
2. 검사 요약
   - 대상 문제 수, 모호표현 발견 수
3. 상세 목록
   - `문항 ID`, `카테고리`, `난이도`, `문항`, `문제 위치`, `발견 토큰`, `권장 보강 기준`
4. 수정 권고
   - 각 문항에 대해 `가장/최초/최대`를 보완한 대체 문안 제안

## 규칙
- 발견 항목이 0개면 `PASS`로 표시.
- 모호 표현이 있으면 해당 항목은 `NEEDS_CLARIFICATION`로 표시하고, 기준 제시 후 수정 요청.
