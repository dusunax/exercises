# quiz-leaderboard

`localStorage`의 순위 데이터를 읽어 리더보드를 조회·분석·관리한다.

## 입력
- `$ARGUMENTS` : 서브커맨드(선택)
  - 비어 있거나 `show` : 전체 카테고리 순위 출력
  - 카테고리명 : 해당 카테고리 순위만 출력
  - `reset <카테고리>` : 해당 카테고리 순위 초기화 (확인 후 실행)
  - `reset all` : 전체 순위 초기화 (확인 후 실행)
  - `analyze` : 점수 분포 및 패턴 분석

## 관련 소스
- `034/src/core/storage.js` — `RANKING_STORAGE_KEY`, `loadRanking`, `saveScoreForCategory`
- `034/src/core/constants.js` — `CATEGORIES`, `RANKING_STORAGE_KEY`
- `034/src/core/scoreManager.js` — 점수 계산 로직 참고

## 스토리지 구조
```json
{
  "여행":     [{ "score": N, "date": "ISO8601", "mode": "category|full|speed" }, ...],
  "음식":     [...],
  "일상":     [...],
  "미술/예술": [...]
}
```
키: `localStorage["quiz-quiz034-rankings-v1"]`

## 실행 규칙

### `show` / 기본
1. 소스 파일에서 `RANKING_STORAGE_KEY` 값을 확인한다.
2. 카테고리별 상위 5개 기록을 표 형태로 출력한다.
3. 전체 1위(가장 높은 점수)를 별도로 하이라이트한다.

### `analyze`
1. 전 카테고리 모든 기록을 수집한다.
2. 다음 항목을 분석한다:
   - 카테고리별 평균 점수 / 최고 점수 / 최저 점수
   - 모드별(full / category / speed) 기록 수
   - 점수대 분포 (0~30 / 31~50 / 51~70 / 71~90 / 91~100+)
   - 가장 활발히 플레이된 카테고리 (기록 수 기준)

### `reset`
1. 대상 카테고리(또는 전체)의 기록 수를 먼저 출력한다.
2. "정말 초기화하시겠습니까? (yes/no)" 확인을 요청한다.
3. `yes`이면 해당 키를 삭제하고 결과를 보고한다.
4. `no`이면 취소 메시지를 출력하고 종료한다.

## 출력 템플릿 (show)

```
=== 034 퀴즈 리더보드 ===

[여행]
순위 | 점수  | 모드     | 날짜
-----|-------|----------|----------
  1  |  95점 | category | 2025-05-01
  2  |  88점 | full     | 2025-04-30
...

[음식] ...
[일상] ...
[미술/예술] ...

★ 전체 최고 기록: [카테고리] [점수]점 ([날짜])
```

## 규칙
- 기록이 없는 카테고리는 `(기록 없음)` 으로 표시한다.
- `reset` 명령은 반드시 확인 단계를 거친 후 실행한다. 확인 없이 삭제하지 않는다.
- 날짜는 `YYYY-MM-DD` 형식으로 표시한다.
