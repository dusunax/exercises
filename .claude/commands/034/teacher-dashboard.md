# teacher-dashboard

선생님 모드의 모든 명령을 순서대로 실행하는 통합 대시보드.
학생 성적 입력 → 비교 → 인사이트 분석 → 리포트 생성 → 내보내기를 한 번에 수행한다.

## 입력
- `$ARGUMENTS` : 실행 모드(선택)
  - 비어 있거나 `full` : 전체 파이프라인 실행 (2~5단계)
  - `report` : 리포트 생성 및 내보내기만 실행 (4~5단계)
  - `analysis` : 비교 및 인사이트만 실행 (2~3단계)
  - `status` : 현재 데이터 현황만 출력 (파이프라인 실행 안 함)

## 명령어 목록

| 명령어 | 기능 |
|---|---|
| `/teacher-input <이름> <점수> [카테고리] [모드]` | 학생 성적 입력 |
| `/teacher-compare [카테고리\|학생명...]` | 학생 성적 비교 |
| `/teacher-insights [학생명\|카테고리]` | 학급 취약점 및 성장 분석 |
| `/create-report [제목]` | HTML 성적 리포트 생성 (상대평가 등급) |
| `/export-report [csv\|pdf]` | 리포트를 CSV 또는 PDF로 내보내기 |

## 실행 파이프라인

### 1단계 — 현황 확인
`034/teacher_scores.json`을 읽어 현재 학생 수와 기록 수를 출력한다.
파일이 없으면 "성적 데이터 없음 — `/teacher-input`으로 먼저 입력하세요"를 출력하고 종료.

### 2단계 — 성적 비교 (`full` | `analysis`)
`/teacher-compare all`과 동일한 동작을 수행한다.
결과를 대시보드 출력에 포함한다.

### 3단계 — 인사이트 분석 (`full` | `analysis`)
`/teacher-insights all`과 동일한 동작을 수행한다.
취약 학생과 취약 카테고리를 하이라이트한다.

### 4단계 — 리포트 생성 (`full` | `report`)
`/create-report`와 동일한 동작을 수행한다.
`teacher_report.html`을 생성하고 경로를 출력한다.

### 5단계 — CSV 내보내기 (`full` | `report`)
`/export-report csv`와 동일한 동작을 수행한다.
`teacher_report.csv`를 생성하고 경로를 출력한다.

## 규칙
- 각 단계 실패 시 오류를 출력하고 다음 단계로 진행한다 (전체 중단하지 않음).
- `status` 모드는 파일 읽기만 수행하고 아무것도 수정하지 않는다.
- 단계별 소요 시간은 표시하지 않아도 된다.
